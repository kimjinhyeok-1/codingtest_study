# [행렬 테두릴 회전하기]

### 1. 접근 방식
 - 사실 어떤 노드에서 시작해도 최저 비용만 산출하면서 연결하면 된다는 생각은 있었음.
 - 대충 0번 노드에서 시작해서 최소거리로 업데이트 하면서 연결해보면 될 것 같다는 느낌적인 느낌만...
 - 그래서 처음에는 다익스트라 느낌으로 연결되어 있는 노드들 찾아다니면서 어떻게 해볼까?하다가 실패함. 오히려 다익스트라에 갇힌듯
 - 그리디라고 했으니까 그냥 단순히 가장 짧은 다리만 연결하면 되지 않을까? 하고 해보다가 예외 케이스에 막힘.

### 2. 핵심 로직
 - 그래프 연결 정보 만들어 놓고, heap에 cost 기준으로 항상 최소 비용 다리가 pop되도록 세팅
 - 방문하지 않은 노드면 해당 다리 확정하고 그 다음에 연결된 노드들을 힙에 추가

### 3. 시간 복잡도
 - O(NlogN)
