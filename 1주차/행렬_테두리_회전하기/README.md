# [행렬 테두릴 회전하기]

### 1. 접근 방식
 - 기존 방향성이 처음에는 board를 정말 굳이 그려야할까? 하는 고민에 시간 좀 녹이다가 결국엔 borad 그려서 푸는걸로 확정
 - rotate하는거 직접 구현해서 풀었음.

### 2. 핵심 로직
 - 돌리는거 각각 x증가, y증가, x감소, y감소 관점에서 각각 구현함.
 - 돌릴 때 모든 수 보고 가장 작은거 보관

### 3. 시간 복잡도
 - O(N^2)

### 4. 느낀 점 / 어려웠던 점
 - x, y 어떤 방향으로 뻗는지 문제 잘 읽을 것.
 - 내가 푼것보다 사실 deque로 푼걸 함께 공유하고픔.
