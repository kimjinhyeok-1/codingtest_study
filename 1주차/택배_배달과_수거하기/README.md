# [택배 배달과 수거하기]

### 접근 방식
 - 일단 n이 10만 -> 완전탐색 불가능
 - 직접 그려보면서 계산해보니 뒤에서부터 차례로 택배를 제거하는 그리디라고 생각함.

### 핵심 로직
 - 배달하는 것과 수거하는 것을 각각 i, j로 두어 각각 인덱스 관리
 - 동일 턴에 인덱스를 비교해서 더 큰쪽으로 최대거리 잡아야 함.

### 시간 복잡도
 - O(N)

### 느낀점/ 어려웠던 점
 - 푸는 방향성 자체는 크게 벗어나지는 않았는데 내가 간과했던 점들이 좀 많았음.
 - 솔직히 그리디 알고리즘으로 풀어야 한다는 생각이 이미 있었어서 예상을 좀 했던 것 같음.
 - 어떤 알고리즘으로 푸는지 모르는 상태였다면 과연 그리디를 떠올릴 수 있었을까? -> 장담 못할 것 같음
 - 문제 풀면서 항상 느끼는거지만 코드 짤 때 마무리가 좀 부족하다는 생각 -> 집중력이 좀 떨어졌나?
 - 정답 코드 확인해보니까 코드를 더 깔끔하게 짤 수 있다는 것을 깨달음. 난 왜 저런 구조화가 안 될까...
