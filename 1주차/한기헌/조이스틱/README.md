# [조이스틱]

### 1. 접근 방식
 - 일단 각 자리별로 알파벳 맞추기 위해 조작하는 횟수와 자리를 이동하는 횟수는 독립이라고 생각해서 분리함.
 - 위아래로 최대 13씩이니까 올리는 경우 내리는 경우 최솟값으로 변환 횟수 지정함.
 - 옆자리로 옮기는 조작을 어떻게 구현할 지 고민을 많이 함.
 - 이건 좌우로 왔다갔다 하는 경우가 있을 수 있어서 완전탐색을 해야하나? 싶었음.
 - 하지만 결국 해결책을 찾아내지 못 함.

### 2. 핵심 로직
 - 아래 코드처럼 현재 확인하는 인덱스를 제외하고 다음 바꿔야 하는 인덱스 저장(next_i)
 - 공백 구간을 체크해서 해당 구간을 제외하고 순회한다는 느낌
 - 1. 현재까지 온 i + 돌아가는 i + 거꾸로 순회할 때 가는 거리(N - next_i)
 - 2. 거꾸로 순회~~~ 
 - 2개를 비교해서 더 값이 작은걸로 확정
```
move = N - 1
    for i in range(N):
        # i를 지난 뒤 연속된 A 구간 끝
        next_i = i + 1
        while next_i < N and name[next_i] == 'A':
            next_i += 1
        
        # 오른쪽으로 i까지 → 왼쪽으로 회차 → 끝 구간
        move = min(move, i + i + (N - next_i))    # 정→역
        move = min(move, (N - next_i) + (N - next_i) + i)  # 역→정
```

### 3. 시간 복잡도
 - O(N)

### 4. 느낀 점 / 어려웠던 점
 - 좌우로 왔다갔다 하면서 횟수를 모두 찾는게 잘 떠오르지 않았음.
 - 단순 거리 개념에 매몰되어 있었던 것 같음.
 - 이게 선형적으로 움직이기 때문에 공백 구간만 접근하지 않으면 된다는 사실을 떠올리지 못 한 것 같음.
