# [큰 수 만들기]

### 접근 방식
 - n이 100만 -> for문 한개만 써서 어떻게 이걸 비교를 하지? 고민함
 - 수를 배열에 담아두고 숫자 비교해서 큰 것을 다시 담는다는 것까지는 생각했음.

### 핵심 로직
 - 스택을 활용한게 포인트
 - 현재 스택의 가장 마지막 인덱스에 있는 수와 지금 받아온 수를 비교해서, 지금 받아온 수가 더 크면 스택을 pop함.
 - 스택이 없거나, 숫자를 빼는 갯수를 다 채우거나, 현재 수가 stack의 가장 윗부분보다 작으면 append 

### 시간 복잡도
 - O(N)

### 느낀점/ 어려웠던 점
 - 문제풀 때 스택을 활용한다는 것을 못 떠올린 순간부터 아마 이 문제는 못 푸는 문제였을 듯.
 - 다른 방법들도 생각해봤는데 n이 100만이라 시간 초과 났을 것.
 - 그리디에서 스택을 활용하는 방법에 대해 배운 것 같음.

### 풀이 시간
 - 한 30분~1시간 고민해봤는데 도저히 안 떠올라서 답 확인.
