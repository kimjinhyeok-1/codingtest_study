# [큰_수_만들기]

### 1. 접근 방식
- 앞 글자부터 확인하면서 greedy로 처리함.

### 2. 핵심 로직
- 총 세가지를 고려. 1. stack에 문자가 있는지. 2. stack 마지막 문자보다 내가 더 큰지 3. k가 0 보다 큰지(삭제 가능 여부)

### 3. 시간 복잡도
- for문 한번 돌기 때문에 O(N)

### 4. 느낀 점 / 어려웠던 점
- 마지막에 while 문 안에 if 문으로 k>0 넣었는데 무한루프에 걸려버림 
- 무한루프 걸리면 반복문 꼭 확인 디버깅 필수

### 5. 풀이 시간
- 20분 가량 품
