# [행렬_테두리_회전하기]

### 1. 접근 방식
- 전형적인 구현 문제로 하나하나 순서대로 시뮬레이션 구현함

### 2. 핵심 로직
- 4개의 변을 기준으로 for 문 돌림

### 3. 시간 복잡도
- O(n^2) n이 작고 구현 문제라 시간 복잡도는 크게 신경 안씀

### 4. 느낀 점 / 어려웠던 점
- 인덱스 잡는게 어려워서 처음에 그냥 1 마이너스 하고 시작함
- r, c 헷갈리고 처음에 덮어써지는게 헷갈렸는데 초기값 arr[x1][y1] 를 따로 저장하여 해결함

### 5. 풀이 시간
- 대략 20분 정도 걸림
